1.7 Backend Logic: Combine All Layers

In Bubble terms, in a backend workflow called for example build_curtain_profile_for_user:

Step 1: Pull components
	•	user
	•	zodiac_profile = Do a search for ZodiacCurtainProfile:first item where zodiac_sign = user's zodiac_sign
	•	archetype_profile = Do a search for ArchetypeCurtainProfile:first item where archetype = user's main_archetype
	•	shadow_profile = Do a search for ShadowCurtainProfile:first item where shadow_type = user's main_shadow_type
	•	energy_profile = Do a search for EnergySubtypeCurtainProfile:first item where energy_subtype = user's energy_subtype

Step 2: Compute baseline numbers

You can store them in temporary custom states or directly in fields on CurtainProfile.

For example:
	•	baseline_open_speed =
`zodiac_profile.base_open_speed
	•	archetype_profile.open_speed_modifier
	•	(energy_profile.role_based_openness - 5) / 2
	•	(user.masculine_polarity_level + user.feminine_polarity_level) / 40`

Then clamp it to 1 to 10.

Same idea with:
	•	baseline_close_speed
	•	baseline_openness
	•	trust_threshold
	•	reopen_difficulty

You do not need to be mathematically perfect on day one. You just need consistent logic you can tune later.
